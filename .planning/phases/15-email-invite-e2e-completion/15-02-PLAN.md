# Plan 15-02: Pending Invites UI + Accept/Decline Flow

## Goal

Build the UI for users to see, accept, and decline pending invites. Also wire `useClaimInvites` into the app layout to auto-claim on login.

**What exists:**
- `group_invites` table with all needed columns
- RLS allows invitees to SELECT their invites (via `auth.email()`)
- `claim_pending_invites` RPC from Plan 15-01
- `useClaimInvites` hook from Plan 15-01

**What's missing:**
- No screen to view pending invites
- No accept/decline UI actions
- No DELETE RLS policy on `group_invites` for invitees (needed for decline)
- `useClaimInvites` is not wired into app lifecycle

### B. Hooks — `usePendingInvites`, `useAcceptInvite`, `useDeclineInvite`

Add to: `src/features/groups/hooks.ts`

```typescript
// INVT-E2E-03: Fetch pending invites for the current user
export function usePendingInvites() {
  return useQuery({
    queryKey: ['invites', 'pending'],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('group_invites')
        .select(`
          id,
          group_id,
          invited_email,
          invited_by,
          accepted_at,
          expires_at,
          created_at,
          groups ( id, name ),
          profiles:invited_by ( display_name )
        `)
        .is('accepted_at', null)
        .gt('expires_at', new Date().toISOString())
        .order('created_at', { ascending: false })
      if (error) throw error
      return data
    },
    staleTime: 30_000,
  })
}

// Accept an invite — calls RPC which adds to group_members + stamps accepted_at
export function useAcceptInvite() {
  const qc = useQueryClient()
  return useMutation({
    mutationFn: async (inviteId: string) => {
      const { data: { user } } = await supabase.auth.getUser()
      if (!user?.email) throw new Error('Not authenticated')
      const { data, error } = await supabase.rpc('claim_pending_invites', {
        target_user_id: user.id,
        target_email: user.email,
      })
      if (error) throw error
      return data
    },
    onSuccess: () => {
      qc.invalidateQueries({ queryKey: ['invites', 'pending'] })
      qc.invalidateQueries({ queryKey: ['groups'] })
    },
  })
}

// Decline an invite — deletes the invite row
export function useDeclineInvite() {
  const qc = useQueryClient()
  return useMutation({
    mutationFn: async (inviteId: string) => {
      const { error } = await supabase
        .from('group_invites')
        .delete()
        .eq('id', inviteId)
      if (error) throw error
    },
    onSuccess: () => {
      qc.invalidateQueries({ queryKey: ['invites', 'pending'] })
    },
  })
}
```

**Note:** `useAcceptInvite` uses the same `claim_pending_invites` RPC which claims ALL pending invites at once. If we want per-invite accept, we'd need a separate `accept_single_invite` function. For MVP, claiming all at once is simpler and matches the sign-up flow. We can refine this during execution if needed.

```typescript
// In RootNavigator, after session check:
const { mutate: claimInvites } = useClaimInvites()

  if (session) {
    claimInvites() // Silently claim any pending invites on login

}, [session])
```
