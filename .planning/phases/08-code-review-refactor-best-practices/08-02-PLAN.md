<task id="4" title="Create cross-platform alert utility (architecture-first)">

### What
8 screens use `Alert.alert` — unreliable on web. **Architecture must be correct now** (single call site for Phase 9 to swap to a toast), even though the web implementation is just `window.alert` for now.

### How
1. Create `src/lib/alert.ts`:
```ts
import { Alert, Platform } from 'react-native'

/**
 * Cross-platform alert wrapper.
 * Phase 8: uses window.alert on web, Alert.alert on native.
 * Phase 9: swap this implementation to a toast component — one-file change.
 */
export function showAlert(title: string, message?: string) {
  if (Platform.OS === 'web') {
    window.alert(message ? `${title}: ${message}` : title)
    return
  }
  Alert.alert(title, message)
}

/** Convenience for error mutations — extracts message from Error. */
export function showError(title: string, error: unknown) {
  const message = error instanceof Error ? error.message : 'An unexpected error occurred'
  showAlert(title, message)
}
```

2. Replace `Alert.alert` in these screens:
   - `groups/[id]/index.tsx` (3 calls)
   - `profile.tsx` (2 calls)
   - `expenses/new.tsx` (1 call)
   - `settlement/new.tsx` (5 calls)

3. **Leave auth screens as-is** — they'll be redesigned in Phase 9.

### Verification
- `pnpm tsc --noEmit` passes
- grep: `Alert.alert` only in auth screens + `ExpenseCard.tsx` (placeholder remind)

</task>

<task id="5" title="Extract shared OAuth cold-start hook">

### What
`sign-in.tsx` and `sign-up.tsx` share identical 14-line boilerplate for WebBrowser warm-up + Linking.useURL() cold-start handler.

### How
1. Create `src/hooks/useOAuthWarmUp.ts`:
```ts
import { useEffect } from 'react'
import * as Linking from 'expo-linking'
import { warmUpBrowser, coolDownBrowser } from '@/lib/platform'
import { createSessionFromUrl } from '@/features/auth/hooks'

/**
 * Handles OAuth cold-start redirect + WebBrowser warm-up.
 * Uses platform adapter — no-op on web, real warm-up on native.
 */
export function useOAuthWarmUp() {
  useEffect(() => {
    warmUpBrowser()
    return () => { coolDownBrowser() }
  }, [])

  const url = Linking.useURL()
  useEffect(() => {
    if (url) createSessionFromUrl(url)
  }, [url])
}
```
   (Uses platform adapter from Plan 08-01 task 4)

2. Replace both auth screen boilerplate blocks with `useOAuthWarmUp()`.
3. Remove `Platform`, `WebBrowser`, `Linking` imports from auth screens.

### Verification
- `pnpm tsc --noEmit` passes
- Auth screens don't import `Platform`, `WebBrowser`, or `Linking`

</task>
