# Plan 08-01: Architecture Audit + Cross-Platform Strategy

## Objective

Enforce correct architectural layering, eliminate cross-layer leaks, implement a structured hybrid cross-platform adapter pattern, and enforce strict feature/component/lib boundaries. This plan addresses the **highest-impact structural issues** that affect every future feature.

**Context decisions applied:** Full restructure, hybrid cross-platform, split persister, haptics guard, platform files next to originals, extract invites, rename to `requireUserId`, rename `export/hooks.ts` → `utils.ts`, strict layering enforcement.

## Tasks

<task id="1" title="Extract shared auth helper (requireUserId)">

### What
`getCurrentUserId()` is copy-pasted identically in 4 files. Extract and **rename** to `requireUserId()` — clearer contract (signals it throws if unauthenticated).

### How
1. Create `src/lib/auth.ts`:
```ts
import { supabase } from '@/lib/supabase'

/** Returns the current authenticated user. Throws if not authenticated. */
export async function requireUser() {
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) throw new Error('Not authenticated')
  return user
}

/** Convenience: returns just the user ID string. Throws if not authenticated. */
export async function requireUserId(): Promise<string> {
  return (await requireUser()).id
}
```

2. In each of the 4 files, **delete** the local `getCurrentUserId` function and replace with:
```ts
import { requireUserId } from '@/lib/auth'
```

### Verification
- `pnpm tsc --noEmit` passes
- grep: `grep -r "getCurrentUserId" src/` returns **0 results**

</task>

<task id="3" title="Split persister into platform files">

### What
`persister.ts` has a ~30-line `buildStorage()` with `Platform.OS === 'web'` branch. Per the hybrid cross-platform strategy (confirmed via industry research), this is a prime candidate for file-level splitting.

### How
1. Create `src/lib/persister.native.ts`:
```ts
import { createMMKV } from 'react-native-mmkv'
import { createAsyncStoragePersister } from '@tanstack/query-async-storage-persister'

const storage = createMMKV()
const asyncStorage = {
  setItem: (key: string, value: string) => { storage.set(key, value); return Promise.resolve() },
  getItem: (key: string) => Promise.resolve(storage.getString(key) ?? null),
  removeItem: (key: string) => { storage.remove(key); return Promise.resolve() },
}

export const persister = createAsyncStoragePersister({ storage: asyncStorage })
```

2. Create `src/lib/persister.web.ts`:
```ts
import { createAsyncStoragePersister } from '@tanstack/query-async-storage-persister'

const asyncStorage = {
  setItem: (key: string, value: string) => { localStorage.setItem(key, value); return Promise.resolve() },
  getItem: (key: string) => Promise.resolve(localStorage.getItem(key)),
  removeItem: (key: string) => { localStorage.removeItem(key); return Promise.resolve() },
}

export const persister = createAsyncStoragePersister({ storage: asyncStorage })
```

3. Replace current `src/lib/persister.ts` with barrel:
```ts
export { persister } from './persister'
```
   (Metro resolves `.native.ts` or `.web.ts` automatically)

### Verification
- `pnpm tsc --noEmit` passes
- `pnpm web` works (localStorage persister loads)
- `pnpm start` works on native (MMKV loads)
- `Platform` import removed from persister files

</task>

<task id="4" title="Implement platform adapter for browser warm-up">

### What
Auth screens have identical 6-line `WebBrowser.warmUpAsync()`/`coolDownAsync()` blocks guarded by `Platform.OS === 'web'`. Create a platform adapter so auth screens never import `Platform` or `WebBrowser` directly.

### How
1. Create `src/lib/platform.native.ts`:
```ts
import * as WebBrowser from 'expo-web-browser'
export const isNativePlatform = true
export const warmUpBrowser = () => WebBrowser.warmUpAsync()
export const coolDownBrowser = () => WebBrowser.coolDownAsync()
```

2. Create `src/lib/platform.web.ts`:
```ts
export const isNativePlatform = false
export const warmUpBrowser = () => Promise.resolve()
export const coolDownBrowser = () => Promise.resolve()
```

3. Create `src/lib/platform.ts` (barrel):
```ts
export { warmUpBrowser, coolDownBrowser, isNativePlatform } from './platform'
```

4. Files live **next to originals** in `src/lib/` (not centralized — matches Expo convention).

### Verification

</task>
