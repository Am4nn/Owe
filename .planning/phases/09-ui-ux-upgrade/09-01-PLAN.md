# Plan 09-01: Design Foundations & Token System

## Objective

Establish the comprehensive design system that every subsequent Phase 9 plan (02–05) will consume. This plan defines color tokens, typography, spacing, border radii, glassmorphism effects, shadows, icon strategy, and reusable base components — ensuring visual consistency across all 31 Stitch screens.

> **Reference:** All visual specifications are sourced from [CONTEXT.md](file:///c:/Aman/Coding-Bamzii/Web%20Development/Owe/artifacts/stitch_owe_design/CONTEXT.md) — the exhaustive audit of all 31 Stitch design screens.

<task id="5" title="Create GlassCard component">

### What
The glassmorphism card is the single most common pattern — used on every screen for balance cards, activity items, member rows, modals, etc. Three variants observed: default, elevated (modals), and dashed ("add new" CTAs).

### How
Create `src/components/ui/GlassCard.tsx`:

- **Props:** `variant` (`'default' | 'elevated' | 'dashed'`), `padding` (number, default 16), `pressable` (boolean — wraps in Pressable with scale-to-0.98 animation), `style`, `children`
- **Default variant:** `glass-card` — `rgba(26,29,46,0.7)` bg, `rgba(255,255,255,0.08)` border, 16px radius
- **Elevated variant:** `glass-card-elevated` — `rgba(37,40,64,0.85)` bg, 20px radius
- **Dashed variant:** `glass-card-dashed` — same bg but `1px dashed rgba(255,255,255,0.15)` border
- **Pressable:** Uses `useAnimatedStyle` from Reanimated for scale press feedback
- On native: use `expo-blur` `BlurView` with `intensity={12}` (fallback to opaque bg if blur unavailable)
- On web: CSS `backdrop-filter: blur(12px)` via style

### Verification
- `pnpm tsc --noEmit` passes
- Component renders correctly on web with blur, on native with BlurView

</task>
