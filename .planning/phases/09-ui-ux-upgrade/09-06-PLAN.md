# Plan 09-06: Deferred Feature Screens (Mock-Powered)

## Objective

Design-complete all deferred feature screens with mock data, ensuring full visual parity with Stitch designs. Backend logic and data wiring deferred to Phase 16. Covers 6 Stitch screens: `insights_updated_navigation`, `qr_code_payment_navigation_fixed`, `subscriptions_theme_fixed`, `admin_dashboard`, `payment_methods`, `help_support`.

> [!IMPORTANT]
> Per ROADMAP success criterion #10: "Features designed during Phase 9 but not implemented must have complete, high-parity designs with data/logic powered by mocks. Full functional implementation will follow in Phase 16."

> **Reference:** [CONTEXT.md §3.19, §3.21–3.22, §3.25–3.27](file:///c:/Aman/Coding-Bamzii/Web%20Development/Owe/artifacts/stitch_owe_design/CONTEXT.md)

### UI/UX Pro Max Strict Guidelines (apply to all tasks)
- **Interactions:** Ensure charts and layout elements remain stable (no layout shifts). Use `cursor-pointer` (web) or `activeOpacity={0.7}` (native) heavily.
- **Accessibility & Contrast:** All data visualization components must use color combinations passing 4.5:1 ratio over the `dark.surface` background.
- **Icons:** Use ONLY Lucide SVG icons with a fixed 24x24 viewBox (`w-6 h-6`). No emoji icons allowed.
- **Glassmorphism:** Strict adherence to `border-white/10` to visually detach overlapping cards from the main canvas without relying solely on shadows.

## Tasks

<task id="1" title="Create centralized mock data file">

### What
All deferred screens need realistic mock data. Centralize in one file for easy swap to real data in Phase 16.

### How
Create `src/lib/mock-data.ts` with:
- **Insights:** Monthly spending totals per category, trend data (6 months), category breakdown percentages
- **Subscriptions:** Netflix/Spotify/Gym entries with member lists, billing dates, amounts
- **Admin:** User count, group count, 24h transaction volume, system health metrics, flagged expenses
- **Payment Methods:** Primary bank, PayPal/Venmo/CashApp connected accounts
- **QR Payment:** User QR code data, username, payment link
- **Support:** FAQ categories, recent articles, search results

All typed with TypeScript interfaces for easy replacement.

</task>
