# Plan 09-05: Feed, Notifications & Polish

## Objective

Build the activity feed, notifications center, and transaction history screens. Also finalize system status screens (error, offline, maintenance). Covers 7 Stitch screens: `activity_feed_updated_navigation`, `activity_feed_with_search_filter`, `notifications_center_updated_navigation`, `transaction_history`, `settlement_success_strict_dark`, `general_error_404`, `no_internet_connection_fixed_nav`, `system_maintenance`.

> **Reference:** [CONTEXT.md §3.14–3.15, §3.20, §3.23, §3.28–3.30](file:///c:/Aman/Coding-Bamzii/Web%20Development/Owe/artifacts/stitch_owe_design/CONTEXT.md)

### UI/UX Pro Max Strict Guidelines (apply to all tasks)
- **Interactions:** Use `Reanimated` spring scale interactions or `activeOpacity={0.7}` on list items and actions. No layout shifts on press.
- **Accessibility & Contrast:** Muted text (timestamps, statuses) must meet 4.5:1 contrast (use `#94A3B8` or `#64748B`). 
- **Icons:** Use ONLY Lucide SVG icons (fixed 24x24 viewBox). No emoji icons.
- **Layout & Spacing:** Avoid strict full-width. Use safe-area edge paddings (`px-4` or `px-6`). Use gap-spacing to separate feed and notification cards instead of raw lines.

### What
New `app/(app)/activity/index.tsx`. The Activity tab in the bottom nav.

### How
**Layout (CONTEXT 3.14):**
- **Header:** search icon left + "Activity Feed" center + settings gear right
- **Filter pills:** SegmentedControl — "All" (active, purple) / "Groups" / "Friends" / "Settlements"
- **Activity cards (stacked GlassCards):**
  - Avatar (40px) left
  - Rich text: **"User Name"** bold + action + **"expense name"** bold + optional "in **Group Name**" purple link
  - Timestamp + category tag (body-sm, secondary)
  - Amount right (AmountText — green positive / red negative)
  - Optional reaction row: thumbs-up, heart, emoji count badges (gray pill-shaped)
- No dividers between cards — gap spacing

**Search variant (CONTEXT 3.15):** Toggle search bar with glass-input at top when search icon tapped.

**Data:** Use existing `useActivityEvents` hook from `features/activity/`. Filter by type based on selected segment.

</task>
